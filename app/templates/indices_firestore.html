Como Criar os Índices

Opção 1: Firebase Console (Recomendado)

  1. Vá para: Firebase Console
  2. Firestore Database → Índices
  3. Clique em Create Index
  4. Preencha com os dados da tabela acima
  5. Espere ~5 minutos por índice

Tempo total: ~20 minutos para 4 índices

Opção 2: Firebase CLI (Mais rápido)

  1. Abra o Terminal na pasta do projeto
  2. firebase deploy --only firestore:indexes --project seu-projeto-id
  3. Espere concluir (~1–2 minutos)
  4. Pronto! Todos os índices criados de uma vez

Tempo total: ~10 minutos

Impacto de Performance

Sem Índices

  • Query com 3 filtros: 2–5 segundos
  • Leitura de documentos: 1000+ ops
  • Custo Firestore: Alto
  • Experiência do usuário: Lenta

Com Índices

  • Query com 3 filtros: 50–200ms
  • Leitura de documentos: 50–100 ops
  • Custo Firestore: 10x menor
  • Experiência do usuário: Rápida

APIs Otimizadas Disponíveis

1. GET /api/chamados/paginar

Descrição: Paginação com cursor — busca apenas 50 documentos

Query params:

cursor=abc123         # ID do último documento (opcional)
limite=50             # Documentos por página (padrão: 50, máx: 100)
status=Aberto         # Filtrar por status
categoria=Projetos    # Filtrar por categoria
gate=G1               # Filtrar por gate
search=termo          # Busca full-text

Response:

{
  "sucesso": true,
  "chamados": [...],
  "paginacao": {
    "cursor_proximo": "id123",
    "tem_proxima": true,
    "total_pagina": 50,
    "limite": 50
  }
}

2. POST /api/carregar-mais

Descrição: Carregar mais registros (infinite scroll)

Body (JSON):

{
  "cursor": "abc123",
  "limite": 20
}

Response:

{
  "sucesso": true,
  "chamados": [...],
  "cursor_proximo": "xyz789",
  "tem_proxima": true
}

3. POST /api/atualizar-status

Descrição: Atualizar status sem recarregar a página

Body (JSON):

{
  "chamado_id": "abc123",
  "novo_status": "Concluído"
}

Response:

{
  "sucesso": true,
  "mensagem": "Status alterado para Concluído",
  "novo_status": "Concluído"
}
